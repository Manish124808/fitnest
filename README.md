# FitNest v2.3

## Render Deploy

1. Push to GitHub, connect repo to Render
2. Build: `cd client && npm install && npm run build && cd ../server && npm install && cp -r ../client/dist public`
3. Start: `node server/server.js`

## Required Env Vars on Render

| Key | Value |
|-----|-------|
| `MONGO_URI` | Your MongoDB Atlas connection string |
| `JWT_SECRET` | Auto-generated by Render |
| `ADMIN_EMAIL` | Your admin email (e.g. `you@gmail.com`) |
| `FRONTEND_URL` | Your Render URL (e.g. `https://fitnest.onrender.com`) |
| `GOOGLE_CLIENT_ID` | From Google Cloud Console (optional) |
| `GOOGLE_CLIENT_SECRET` | From Google Cloud Console (optional) |
| `GOOGLE_REDIRECT_URI` | `https://your-app.onrender.com/api/googlefit/callback` |

## Admin Panel Fix

**If you already registered and admin panel shows blank / access denied:**

The server now **auto-grants admin** to `ADMIN_EMAIL` on every startup.
So just:
1. Set `ADMIN_EMAIL` in Render env vars = your login email
2. Redeploy (or restart) — server will auto-set `isAdmin: true` in MongoDB
3. Login normally → Admin tab will appear

Alternatively, use the **Tools tab** in Admin Panel → Promote User.

## Seed Food Database

After deploy, run once to seed Indian + hostel mess foods:
```
node server/scripts/seed.js
```
Or via Render Shell: `node scripts/seed.js`

## Google Fit

`googleapis` is in `package.json` and auto-installs on Render.
No manual `npm install googleapis` needed.
